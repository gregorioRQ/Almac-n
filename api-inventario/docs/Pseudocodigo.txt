REGISTRO USUARIO
Controlador de registro recibe el dto con los datos del usuario
Genera un token valido con los datos.

CREACION DEL PERFIL
El usuario autenticado envia un dto con su tipo de rol
mediante un dto.
UserService edita el valor "rol" de la entidad y crea una nueva entidad (Proveedor/vendedor) y le asigna el usuario editado.


Mecanismo de descuento:
(se supone que el vendedor comprara por lotes)

recibir dto Descuento(id item, atributos que detallen cantidad a descontar);
buscar el item en la db segun el dto.
segun el dto modificar los valores referidos a la cantidad.
condicional: si ahora la cantidad es menor a x numero notificar al proveedor

FUNCIONAMIENTO DEL SISTEMA.

OBTENCION Y GUARDADO DEL ITEM
Proveedor entrega los datos del item que desea guardar mediante un formulario a logistica.

Websocket notifica  de un nuevo item, con la intención de "ser guardado" a 
los roles de Logistica para que todo el departamento tenga el registro de seguimiento del item.

logistica de transporte al igual que los demas roles de logistica, recibe la notificacion y sale rumbo a recoger el itme, luego cambia el estado del itam a "en camino".

cuando el item llega a la bodega logistica de transporte notifica a logistica de repositorios
mediante websokcet.

Logistica almacena el nuevo item.
(usuario que pertenece al departamento de logística con el rol "repositor").

El item tiene que tener un regitro de quien/quienes lo transportaron.

Añadir a item la prop "manipuladoPor".

ItemService modificar y guardar el item.

el estado del item pasa a ser guardado y se notifica de ello al proveedor.

DESPACHAR UN ITEM
Logistica recibe notificación para despachar x item.
(No elimina la entidad)


PROVEEDOR
cuando la cantidad de lotes llegue a 0, notificar al proveedor que a partir de ese momento cuenta con x cantidad de dias para volver a reponer su item sino sera eliminado de la base de datos.

Si item es eliminado notificar a proveedor.
